第一阶段：先把“能安全用”做出来（2-3 周）
1.统一身份与权限
2.所有业务 API 强制鉴权（Bearer Token）
3.引入 RBAC：普通用户、管理员，后续可扩展组织管理员
4.给所有核心数据表加 owner_id（或 tenant_id + owner_id）
5.所有查询都按 owner_id/tenant_id 过滤，禁止跨用户读取
6.验收标准：A 用户看不到 B 用户任何项目/文档/笔记/高亮数据

第二阶段：文档管理系统生产化（2-4 周）
1.建立文档元数据模型：文档、版本、标签、状态、权限
2.文件存储改为对象存储（S3/OSS/MinIO），数据库仅存元数据和访问策略
3.上传流程改为预签名 URL + 后端回调确认
4.增加文档版本控制、软删除、回收站、恢复
5.做全文检索（先标题/摘要/标签，后续再加向量检索）
6.验收标准：文档上传、更新、权限、版本回滚可追踪且稳定

第三阶段：全量活动数据留痕（1-2 周）
1.新增审计事件表（append-only，不可修改）
2.记录关键事件：注册、登录、登出、上传、下载、创建/编辑/删除、权限变更、支付操作
3.每条事件至少含：who、when、what、resource、ip、ua、request_id
4.前端行为埋点与后端审计分离：前端用于产品分析，后端用于合规审计
5.验收标准：能按用户、时间、资源追溯完整操作链

第四阶段：替换演示逻辑，彻底去 mock（1-2 周）
1.清理 localStorage 作为主存储的路径，只保留短期缓存
2.去掉 mock PDF 内容和演示假数据入口
3.初始化脚本改为“可控种子数据”，仅在 dev/staging 启用
4.验收标准：刷新页面/换设备后，业务数据一致且来自后端

第五阶段：生产部署与运维（1-2 周）
1.数据库切 PostgreSQL（不要再用 SQLite 作为生产主库）
2.增加 Redis（会话、限流、任务队列）
3.采用容器化部署（Docker + Nginx + HTTPS + CI/CD）
4.配置监控告警：错误率、接口延迟、队列堆积、磁盘、CPU、内存
5.日志集中化（结构化 JSON），支持按 request_id 全链路排查
6.备份与恢复演练：RPO/RTO 明确并演练通过

第六阶段：安全与合规（并行推进）
1.密钥管理：环境变量/密钥服务，不落库不入日志
2.输入校验、上传白名单、病毒扫描、限流、防爆破
3.CSRF/CORS/安全响应头策略完善
4.数据脱敏与最小化收集，准备隐私协议与用户条款
5.上线前做一次渗透测试和依赖漏洞扫描

建议的技术底座（你现在就可以定）
1.后端：FastAPI + PostgreSQL + SQLAlchemy + Alembic + Redis + Celery/RQ
2.前端：React + 统一 API 层 + React Query（或 SWR）+ 严格错误边界
3.存储：S3/OSS/MinIO + 预签名上传
4.观测：Sentry + Prometheus/Grafana + Loki/ELK
5.CI/CD：GitHub Actions（测试、构建、迁移、部署）

你接下来“本周就要做”的 8 件事
1.冻结数据模型：用户、项目、文档、审计事件四大核心表
2.明确权限矩阵（谁能看、谁能改、谁能删）
3.把所有业务接口接入鉴权并加 owner/tenant 过滤
4.前端 API 改为后端优先，localStorage 只做缓存兜底
5.上 PostgreSQL，完成第一版迁移脚本
6.增加审计事件中间件（每个请求打 request_id）
7.搭建 staging 环境，跑真实注册-登录-上传-检索-删除流程
8.设定上线门槛：测试覆盖率、P95 延迟、错误率、备份恢复演练通过
